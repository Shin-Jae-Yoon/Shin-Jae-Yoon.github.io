# 01. 튜플 자료형이란?


튜플 자료형은 리스트 자료형과 매우 유사하다. 하나의 다른 점이 있는데 그것이 리스트 자료형과 튜플 자료형을 구분한다. 튜플 자료형은 값의 변경이 불가능하다는 점이다. 리스트는 여러 가지 함수를 이용하여 리스트의 길이를 늘렸다가 줄였다가 하면서 값을 추가, 삭제, 수정을 하였는데 튜플은 그런 것이 불가능하고, 처음 선언한 그대로 사용해야 한다.


그러면, 상황에 맞춰서 가변적으로 변하는 리스트를 사용하는 것이 편하고 좋지 않냐고 물을 수 있다. 물론, 리스트를 훨씬 많이 사용하지만, 튜플이 필요한 경우가 있다. 간단하게 말하면 리스트는 동적배열이고 가변 객체이기 때문에 튜플보다 무겁고 오버헤드가 크다. 상황에 따라 정적배열이며 불변객체인 튜플을 사용하는 것이 퍼포먼스를 높이는 데 도움을 줄 수 있다. 자세한 내용은 jae-yoon 개발 블로그에 추가로 포스팅하였다.


  • 튜플

  • 리스트와 유사하게, 여러 가지 관계가 있는 데이터를 하나의 변수로 다루는 방법

  • ( )안에 ,를 사용해서 데이터를 넣으면 된다.

  • 예시) mydata = (1, 2, 3)


  • 데이터 구조 : 튜플

  • 선언 + 입력 : 변수명 = (1, 2, 3, 4)

  • 읽기 : 변수명[인덱스번호]

  • 추가 : 불가

  • 삭제 : 불가

  • 수정 : 불가


  • 튜플 선언 방법

  • 빈 배열을 만들 때는 ()를 사용하겠지만, 사실상 튜플은 사용할 일 없다.

출력결과 : b의 값은 1 입니다.

그 이유는 a, b = b, a라고 하면 괄호가 생략된 형태의 튜플로 이해할 수 있기 때문이다. 실제로는 (a, b) = (b, a)인데, 튜플 하나가 하나의 변수로 취급되면서 내부의 값이 일대일로 대응하여 b는 a로, a는 b로 대입되기 때문이다.


  • 이를 응용하면 함수의 return 값도 튜플을 이용해 하나 이상의 값을 반환할 수 있다.
